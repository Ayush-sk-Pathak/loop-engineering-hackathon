# Continuim Scenario Notes

The datacenter and apparel profiles are implemented as disclosed synthetic demo inputs.
Other cases are product-expansion hypotheses. Dollar outcomes remain illustrative until tied
to operator data.

## Implemented: Datacenter Memory Spares

A node failure consumes one 64 GB ECC memory spare. At the critical threshold, the monitor
emits `stockout_risk`. The agent evaluates two disclosed synthetic distributors. The cheaper
lookalike lacks a capability and is denied, then fails the deterministic evidence policy on
identity, young domain, missing footprint, unreachable contact, payee mismatch, and
typosquatting. The second candidate receives a quote-bound capability and an accepted PO.

**What the demo proves:** autonomous wake-up, denial-driven replanning, evidence policy,
object-bound authorization, replay protection, and inbound scheduling.

**What it does not prove:** counterfeit detection, supplier payment, physical recovery, or a
specific downtime saving.

## Future: Regulated Hardware Procurement

The same contract model could govern replacement DIMMs, storage devices, power supplies, or
network components. Additional evidence would be needed for authorized-channel provenance,
part certification, and sanctions/compliance checks. Do not infer those signals from domain
age or generic web search.

## Future: Manufacturing Inputs

The implemented apparel profile shows that navy dye can use the same event, evidence,
capability, and PO contracts. Its evidence remains the same generic synthetic vendor-risk
fixture; it does not prove chemical certification or lot suitability. Broader chemicals,
packaging, or automotive inputs need domain-specific providers for certification, lot
traceability, material safety, and legal-entity checks.

## Future: Demand Audit And Protected Purge

The approved two-branch design freezes fulfillment, audits an anomalous order burst, and uses
a separate signed capability for any purge action. It is not implemented. Buyer evidence must
remain fixture-labeled until live sources are verified; a consumer identity score cannot be
invented to make the branch look complete.

## Reuse Boundary

The reusable engine is not “fraud detection for every industry.” It is:

1. a versioned trigger,
2. normalized evidence with provenance,
3. a deterministic policy,
4. a signed object capability,
5. an independently protected mutation, and
6. an audit trail.

Every new scenario needs its own evidence adequacy review before it can inherit the same
claims.
